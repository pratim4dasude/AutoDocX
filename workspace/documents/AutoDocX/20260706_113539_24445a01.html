Overview

This project provides AutoDocX, a developer-documentation engine that scans a local Python project, extracts structured code metadata, generates an LLM-backed human-readable understanding, and produces standalone HTML developer documentation. It exposes a FastAPI HTTP API for scanning, comparing, building LLM context, generating understandings, and saving/serving documentation artifacts, plus a Streamlit UI client for interactive use.
API paths, handlers, signatures, parameters, classes, and methods are taken from parser data where available, not guessed by the LLM.

Architecture

The application uses a layered architecture: (1) HTTP layer (app.main and app.api.project_routes) exposes synchronous REST endpoints for scanning, comparison, LLM understanding and document generation; (2) Service layer (app.services.*) implements core logic: ProjectScanner (file discovery, hashing, parse invocation), PythonCodeParser (AST extraction), ProjectAnalyzer (project-level relationships and API route assembly), ProjectContextBuilder (compact context generation), ProjectUnderstandingService (LLM orchestration and merging of parsed data), ScanComparator (scan diffs), DocumentBuilder (HTML generation) and storage classes (ScanStorage, UnderstandingStorage, DocumentStorage) for persistence in workspace/*; (3) Provider layer (app.services.llm.*) contains LLM adapters and a factory to create provider instances; (4) UI layer (ui.streamlit_app) is a convenience client that calls the HTTP API. Runtime data moves from scanner -> parser -> analyzer -> context builder -> LLM provider -> understanding merger -> document builder -> document storage; API endpoints orchestrate these flows and storage classes persist intermediate and final artifacts under workspace paths configured in app.core.config.
