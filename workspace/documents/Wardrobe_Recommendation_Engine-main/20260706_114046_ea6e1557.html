Overview

This project provides a Wardrobe Recommendation Engine implemented in Python. It builds metadata and embeddings for wardrobe images, supports hybrid text+visual retrieval, offers reranking with LLMs, stores vectors and conversation memory in PostgreSQL/pgvector, and includes CLI scripts and demos to run ingestion, searches and an interactive chat assistant.

Architecture

The code is organized into utility, ingestion/captioning, embedding, retrieval, reranking, recommender and persistence layers. Utilities (src.utils) provide filesystem helpers. Captioner (src.engine.captioner) turns images into captions/metadata; embeddings (src.engine.embeddings and src.engine.visual_embeddings) create text and visual embeddings and produce hybrid items. Hybrid retrieval and similar-search modules (src.engine.hybrid_retrieval, src.engine.similar_search, src.engine.pgvector_similar_search) perform nearest-neighbour scoring using either in-file hybrid JSON or PostgreSQL-backed vectors (src.db.pgvector_store). LLM-based rerankers (src.engine.llm_reranker, src.engine.llm_similar_reranker) refine candidate lists. Outfit construction and presentation are handled by outfit_recommender and fashion_assistant modules. Chat memory is persisted through a Postgres-backed adapter (src.db.postgres_chat_memory) with a local adapter wrapper (src.engine.chat_memory). Top-level scripts and main.py orchestrate setup, data-building, migrations and demo/chat runs.
